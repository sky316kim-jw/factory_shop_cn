# 프로젝트: factory_shop_cn
중국 공급업체와 한국 바이어 간의 B2B 샘플 발주 관리 웹앱

## 개발자 정보
- 나는 코딩을 전혀 모르는 비개발자야
- 모든 설명은 7살 아이에게 하듯이 쉽게 해줘
- 에러나면 원인과 해결방법을 단계별로 설명해줘

## 앱 핵심 기능
1. 공급업체(중국)가 샘플 상품 등록 (사진+정보)
2. 바이어(한국/나)가 갤러리로 상품 보고 선택
3. 선택 시 수정 코멘트 달아서 발주 확정
4. 발주 시 내부품번 자동생성 (PRD-YYYYMM-XXXX)
5. 발주 상태 추적 (발주완료→생산중→선적완료→입고완료)

## 사용자 역할
- supplier: 상품 등록 전용
- buyer: 상품 열람 + 발주
- admin: 전체 관리

## 기술 스택
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Supabase (DB + Auth + Storage)

## 코딩 규칙
- 모든 코드에 한국어 주석 필수
- 모바일 반응형 필수
- 에러메시지는 한국어로
- 코드는 최대한 단순하게
- 환경변수는 .env.local 에서 관리

## 데이터베이스 주요 테이블
- users: 회원정보 (역할 포함)
- suppliers: 공급업체 정보
- products: 샘플 상품
- product_images: 상품 이미지
- purchase_orders: 발주서
- po_items: 발주 상품 목록
- internal_skus: 내부품번 매핑

## 개발 순서
Phase 1: 로그인 + 공급업체 상품등록 + 바이어 상품목록
Phase 2: 발주 + 내부품번 자동생성 + 발주상태 추적
Phase 3: AI 번역/요약 + 관리자 대시보드

## 중요한 규칙
- 나는 코딩을 전혀 모른다
- 뭔가 물어볼 게 있으면 묻지 말고 그냥 알아서 최선의 방법으로 진행해라
- Yes/No 질문도 그냥 Yes로 알고 진행해라
- 중간에 멈추지 말고 끝까지 완성해라
- "Do you want to proceed?" 같은 질문도 하지 마라
- 진행 여부를 묻는 모든 질문은 그냥 Yes로 알고 계속 진행해라
- 어떤 상황에서도 멈추지 말고 끝까지 혼자 다 해라
